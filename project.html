Project Information

Diffusion LLMs for
Code Identifier Renaming

Explore how discrete diffusion language models can be applied to the software engineering task of variable and method identifier renaming — improving code readability and maintainability automatically.

Project Overview

This project investigates the application of Discrete Diffusion Language Models (dLLMs) to the task of code identifier renaming in Java source code. Poor identifier naming is one of the most widespread forms of technical debt, leading to code that is difficult to understand, maintain, and extend.

Traditional approaches rely on autoregressive models (e.g., CodeBERT, CodeT5) or rule-based heuristics. This project asks: can a non-autoregressive, diffusion-based approach achieve competitive or superior results?

Research Question

RQ
"To what extent can discrete diffusion language models rename poor code identifiers in Java source code, compared to autoregressive and fill-in-the-middle baselines?"

Methodology

1. Bad Naming Detection

Use a multi-probe detector (overconfidence + context sensitivity) to identify poor identifier names in Java code. Indicators include single-letter names, dictionary mismatches, and low contextual coherence.

2. Contextual Masking

Extract code context around the identified bad-name token. Mask the target identifier to enable the model to infer a better replacement using surrounding code semantics.

3. Diffusion Inference

Apply the dLLM (e.g., MDLM, MDLM-GPT, or similar) to iteratively refine the masked token via a forward-backward diffusion process, producing a renaming suggestion.

4. Evaluation

Evaluate using Exact Match (EM), BLEU, and LLM-as-judge metrics against ground-truth renamings from real developer commits. Compare against FIM-based and autoregressive baselines.

Datasets

Primary

Diffusion Benchmark

A curated dataset of Java variable renaming instances extracted from real-world developer commits. Used as the main evaluation corpus for diffusion-based models.

Java Variable Renaming Ground-truth Labels
Baseline

FIM / AR Benchmark

A fill-in-the-middle style benchmark used to evaluate autoregressive and FIM-based baseline models. Enables controlled comparison with the diffusion approach.

Java FIM Format Multi-model
Extended

Developer Dataset

An extended dataset focusing on real developer refactoring sessions for code variable name improvements. Provides additional evaluation diversity.
