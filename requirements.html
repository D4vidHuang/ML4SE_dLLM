Student Requirements

What I Expect
From You

A clear and transparent overview of expectations, deliverables, grading, and conduct policies for students enrolled in ML4SE.

Who Should Choose This Project

⚡
This is an advanced research project. Due to the technical depth involved — working with large language models, diffusion processes, and code analysis — we encourage students with prior experience in the following areas to select this project:
★
Machine Learning & Deep Learning

Comfortable implementing and training neural networks. Familiar with PyTorch or similar frameworks at more than a tutorial level.

★
Natural Language Processing

Some exposure to transformer architectures, tokenization, and language model fine-tuning (e.g., have used Hugging Face before).

★
Software Engineering & Code Analysis

Understanding of source code structure (ASTs, token-level representations). Experience parsing or analyzing code programmatically is a plus.

★
Empirical Research Skills

Ability to design controlled experiments, report results rigorously, and write a structured research report. Previous coursework or project involving evaluation is preferred.

Students who do not fulfil these prerequisites are welcome to reach out to discuss whether this project is the right fit. We are happy to advise.

General Expectations

This is a research-oriented course. You are expected to behave and think like a junior researcher — not just a student completing assignments. This means engaging critically with the literature, making informed decisions about your implementation choices, and expressing your own opinions about the methods.

✓
Active Engagement

Attend all scheduled meetings and seminars. Come prepared with questions and updates on your progress.

✓
Independent Problem Solving

Attempt to troubleshoot technical issues yourself first, using documentation and prior research. Reach out to the TA with specific, well-articulated questions.

✓
Proactive Communication

If you are falling behind or encounter a blocker, communicate early. Do not wait until the deadline to flag problems.

✓
Collaborative Spirit

While you work individually, you are encouraged to discuss ideas with peers. Respect intellectual boundaries — do not share or copy code or analysis.

Technical Requirements

Your implementation must meet the following technical standards. These ensure reproducibility and scientific rigor.

Code Quality
✓
Version Control

All code must be maintained in a private Git repository (GitHub or GitLab). Share access with the TA at the start of the project.

✓
Reproducibility

Your experiments must be fully reproducible. Provide a README with setup instructions, dependency lists, and commands to reproduce results.

✓
Code Clarity

Write clean, well-commented code. Avoid unnecessarily complex implementations — clear is better than clever. Use meaningful variable names (practicing what you study!).

✓
Environment Management

Use a virtual environment (conda or venv). Include an environment.yml or requirements.txt file in the repository root.

Experiment Standards
✓
Baseline Comparison

Always compare against at least one established baseline. Your results only have meaning in context.

✓
Statistical Reporting

Report metrics with appropriate precision (2–3 decimal places). Where applicable, include confidence intervals or standard deviations over multiple runs.

✓
Error Logging

Document failed experiments and what you learned from them. Negative results are scientifically valuable; ignoring them is not.

Final Report Requirements

The final report is the primary graded artifact. It should read as a self-contained research paper, written in English, formatted in IEEE two-column format, and being a minimum of 8 pages (excluding references).

Required

Abstract

A 150–250 word summary of the problem, methods, results, and conclusions. Write this last.

Required

Introduction

Motivate the problem, state the research question clearly, and briefly preview your approach and findings.

Required

Related Work

Cover at least 10 relevant papers. Group by theme. Situate your work in the existing landscape — don't just list papers.

Required

Methodology

Describe your approach in detail. Include diagrams where appropriate. Someone should be able to replicate your work from this section alone.

Required

Experimental Setup

Describe datasets, baselines, metrics, hardware, and hyperparameters used in your experiments.

Required

Results & Discussion

Present results in tables and figures. Analyze them critically — what works, what doesn't, and why?

Required

Conclusion

Summarize findings, acknowledge limitations honestly, and propose future directions.

Bonus

Ablation Study

Systematically vary one aspect of your approach at a time to understand what matters most.

Academic Integrity

TU Delft takes academic integrity extremely seriously. All submitted work must be entirely your own. This includes code, text, and experimental results.

✓
Allowed
  • Using open-source libraries (with proper attribution)
  • Discussing ideas with peers at a conceptual level
  • Using LLMs (ChatGPT, Copilot) as tools, with disclosure
  • Referencing and building on published code (with citation)
✗
Not Allowed
  • Copying code from peers, even partially
  • Using LLMs to write substantial sections of your report without disclosure
  • Fabricating or selectively omitting experimental results
  • Sharing your code or results with other students before submission

When in doubt, ask. It is always better to ask permission than to face consequences later.
